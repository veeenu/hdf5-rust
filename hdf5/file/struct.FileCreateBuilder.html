pub fn istore_k(&mut self, ik: u32) -> &mut Self[src]

Sets the size of the parameter used to control the B-trees for indexing chunked datasets.

istore_k is one half the rank of a tree that stores chunked raw data. On average, such a tree will be 75% full, or have an average rank of 1.5 times the value of istore_k.

The HDF5 library uses istore_k * 2 as the maximum number of entries before splitting a B-tree node. Since only 2 bytes are used in storing the number of entries for a B-tree node in an HDF5 file, istore_k * 2 cannot exceed 65536.

The default value for istore_k is 32.
